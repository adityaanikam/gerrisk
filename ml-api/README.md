# GeriRisk ML API

Standalone FastAPI service that wraps the GeriRisk scikit-learn models and
exposes them over HTTP. Intended to be deployed on **Render** so that the
Next.js app on Vercel can call `POST /predict` instead of spawning a local
Python subprocess (which Vercel does not support).

---

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/health` | Liveness probe — returns `{"status": "ok"}` |
| `POST` | `/predict` | Run all three risk models |
| `GET` | `/docs` | Swagger UI (auto-generated by FastAPI) |
| `GET` | `/redoc` | ReDoc UI |

### `POST /predict`

**Request body**

```json
{
  "avgHeartRate": 78,
  "maxHeartRate": 120,
  "minHeartRate": 55,
  "minSpO2": 94,
  "totalSteps": 3200,
  "recordCount": 48
}
```

All fields are optional and fall back to safe clinical defaults if omitted or
`null`.

**Response**

```json
{
  "cardiacRisk":     { "score": 0.312, "level": "Low" },
  "fallRisk":        { "score": 0.651, "level": "Moderate" },
  "respiratoryRisk": { "score": 0.142, "level": "Low" }
}
```

## Local development

```bash
cd ml-api

# Create a virtual environment
python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate

pip install -r requirements.txt

# Start the dev server (auto-reload on file changes)
uvicorn main:app --reload --port 8000
```

Then open <http://localhost:8000/docs> for the interactive API docs.

Set `ML_API_URL=http://localhost:8000` in `geririsk-ai/.env.local` so the
Next.js app calls this local service during development.

---

## Render deployment

1. Push the monorepo to GitHub.
2. In the Render dashboard → **New → Blueprint** → select your repo →
   Render will detect `render.yaml` at the repo root and create the service.
3. Set the `ALLOWED_ORIGIN` environment variable to your Vercel URL:
   ```
   https://geririsk.vercel.app
   ```
4. After the first deploy, copy the Render service URL (e.g.
   `https://geririsk-ml-api.onrender.com`) and paste it into Vercel as
   `ML_API_URL`.

## Models

Pre-trained scikit-learn models live in `models/`. They are loaded **once**
at startup and kept in memory for the lifetime of the process.

| File | Used for |
|------|----------|
| `cardiac_risk_model.pkl` | Cardiac stress classifier |
| `cardiac_scaler.pkl` | StandardScaler for cardiac features |
| `fall_risk_model.pkl` | Fall likelihood classifier |
| `fall_scaler.pkl` | StandardScaler for fall features |
| `respiratory_risk_model.pkl` | Respiratory risk classifier |
| `respiratory_scaler.pkl` | StandardScaler for respiratory features |

## Environment variables

| Variable | Default | Description |
|----------|---------|-------------|
| `ALLOWED_ORIGIN` | `*` | Comma-separated CORS origins, or `*` to allow all |
| `PYTHONUNBUFFERED` | `1` | Set by render.yaml; ensures real-time logs |
| `PORT` | `8000` | Injected by Render automatically |
